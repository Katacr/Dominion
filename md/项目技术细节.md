## 2. 资源结构与清单

### 2.1 目录职责

```text
core/src/main/resources/uis/
├── manifest.yml
├── _shared/
│   └── <menu-id>.yml
├── en_US/
│   ├── text_menu/
│   ├── chest_menu/
│   └── dialog_menu/
└── zh_CN/
    ├── text_menu/
    ├── chest_menu/
    └── dialog_menu/
```

- `_shared/<menu-id>.yml` 保存 route 参数、provider 映射和 `action-groups`。
- 三种类型目录只保存本地化文本与界面表现。
- 业务目标、权限判断和数据库写入不放入表现层 YAML。
- 所有配置当前使用 `schema-version: 1`。
- YAML 映射使用 block style；`{entry.*}`、`{route.*}`、`{page.*}` 是运行时占位符。

### 2.2 manifest.yml

`manifest.yml` 是 JAR 内置资源清单：

- `locales` 只列随 JAR 发布的语言。
- 三类 menu 列表分别决定默认文件安装和加载范围。
- 外部新增语言不写入 manifest；仓库会扫描 `plugins/Dominion/uis/<locale>/`。
- `action_demo` 明确属于管理员诊断资源。
- 默认资源通过 `saveResource(..., false)` 安装，不覆盖管理员已有文件。
- 新增或删除菜单时必须同步修改 `_shared`、所有内置语言目录、manifest 和资源加载测试。

## 3. 运行时初始化

入口为 `ConfiguredTuiManager`：

```text
ActionRegistry + DominionActionRegistry
        ↓
MenuDataRegistry
        ↓
TuiMenuRepository.load()
        ↓
UiCallbackSessionManager + ActionExecutor
        ↓
TuiRenderer / CuiRenderer / Dialog adapter
        ↓
UiResultRouter + callback commands + listeners
```

- `TuiMenuRepository.load()` 先解析并校验全部资源，成功后才原子替换 snapshot。
- 每次成功加载递增 revision，并清理旧 revision 的 callback token。
- TUI、CUI、DUI 都从同一 snapshot 读取本地化定义。
- CUI listener 只处理 `CuiInventoryHolder` 拥有的 Inventory。
- Dialog adapter 通过反射按平台延迟加载，core 不暴露平台 Dialog 类型。
- 插件关闭时清理 Dialog adapter 和全部 callback session。

平台选择：

```text
Paper 且存在 Paper Dialog API
  -> PaperDialogPlatformAdapter

非 Paper 且存在 PlayerCustomClickEvent 与 Bungee Dialog API
  -> SpigotDialogPlatformAdapter

其他平台或旧版本
  -> DUI 不可用，TUI/CUI 保持可用
```

## 4. 共享配置契约

### 4.1 Route

`MenuRoute` 包含 `menuId`、`page` 和服务端构建的 `arguments`。共享文件声明必填和可选 route 参数：

1. 所有必填参数必须存在。
2. 不允许未声明参数。
3. 翻页保留原 route 参数。
4. Provider 修正越界页后，以有效页码注册 callback。

route 参数决定领地、成员、权限组或模板等页面上下文，客户端不能直接提交 route。

### 4.2 Data source

共享文件将本地 source 名映射到固定 provider ID：

```yaml
data:
  entries: flags.environment
```

当前 provider 覆盖：

- 领地管理树、全部领地、指定玩家领地。
- 环境、访客、成员、权限组和模板 Flag。
- 成员、候选玩家、权限组成员和未分组成员。
- 模板列表、模板选择和复制来源。
- 称号、Residence 迁移和可用语言。

`MenuEntry` 分离：

- `displayVariables`：只用于展示。
- `trustedArguments`：只保存在服务端 session。
- `availableActionGroups`：决定该 entry 可以注册哪些动作。

renderer 的隐藏或禁用只负责体验；业务 action 必须再次鉴权。

### 4.3 Action groups

可复用业务动作写在共享文件：

```yaml
action-groups:
  manage:
    actions:
      - "dominion: manage-dominion"
```

表现层通过 `action-group: manage` 引用。局部反馈可直接配置 `actions`，二者严格二选一。旧 `operation/operations` 已移除。

## 5. Action 系统

### 5.1 动作与注册

通用动作：

- `tell`、`actionbar`、`title`、`toast`、`hovertext`
- `open`、`page`、`refresh`、`close`

Dominion 业务动作统一使用 `dominion: <action-name>`，由显式白名单 registry 分发。registrar 按主菜单、领地树、Flag、成员、权限组、模板、复制和工具操作拆分，不允许 YAML 反射调用 Java 方法。

### 5.2 解析与校验

- YAML 在加载期解析成不可变 `ActionSpec`。
- 未注册动作、重复参数、未知字段和非法控制流会阻止 reload。
- 参数 parser 支持单引号、双引号、反引号和转义。
- `title`、`toast` 使用 `key=value;key=value` 参数。
- 变量命名空间包括 `player.*`、`menu.*`、`route.*`、`page.*`、`entry.*`、`input.*`。
- 动态值以 literal/unparsed 方式插入 MiniMessage，不能注入 click 或 hover 标签。
- 未知占位符保持原样，便于发现配置错误。

### 5.3 执行与结果

`ActionExecutor` 严格顺序执行：

- 只有 `CONTINUE` 继续下一项。
- `STOP`、`OPEN`、`REFRESH`、`CLOSE`、`FAILURE` 终止序列。
- workflow 默认最多 30 秒。
- handler 异常、null future 或 null result 都转换为失败。
- action 从玩家所属主线程或 Folia entity scheduler 开始。
- 已启动的异步业务 future 无法在超时后强制回滚副作用。

`UiResultRouter` 按 callback 保存的 `UiSurface` 返回原 renderer：

- TUI 直接打开或刷新。
- CUI 和 DUI 延迟 1 tick 打开或刷新。
- `CLOSE` 才显式关闭。
- CUI/DUI 普通回调不先关闭再重开，避免闪烁和鼠标回到屏幕中心。

## 6. Callback 与并发安全

### 6.1 Session token

`UiCallbackSessionManager` 使用 32 位十六进制随机 token。每个 callback 保存玩家 UUID、surface、route、revision、action、变量、可信参数、可选 `InputSchema` 和过期时间。

- token 默认有效 5 分钟，只能成功消费一次。
- 跨玩家提交不会消费原玩家 token。
- 新 session、reload、退出和定时清理会失效旧 token。
- 客户端只获得 token，不获得权限路径、数据库 ID 或业务 action。
- 二级确认的 `ui.confirmed` 由 adapter 在服务端加入。

### 6.2 Transport

- TUI：`/dominion ui_callback <token>`。
- CUI：`slot + click type -> token`。
- Spigot Dialog footer：`CustomClickAction` additions。
- Spigot Dialog Body：普通动作使用 `ui_callback`，二级表单使用 `ui_dialog_callback`。
- Paper Dialog：Adventure callback 消费同一服务端 token。
- CUI 关闭只失效该 holder 的 token，不清理后来打开的新菜单。

### 6.3 异步渲染

`UiRequestTracker` 防止较慢的旧 provider 请求覆盖新 route。render 前检查玩家在线、revision 未变化、request ID 仍为最新。

## 7. TUI

TUI 支持 `BUTTON`、`URL`、`COPY`、`LIST`。

主要限制：

- 每个菜单最多一个 LIST。
- LIST `rows` 为 1-20，placeholder 必须独占一行。
- Layout 最多 64 行，每行最多 32 个按钮引用。
- 按钮最多 128 个，变量最多 256 个。
- 固定工具栏不占动态分页容量。

`TuiRenderer` 将 layout、静态按钮和 provider entries 组合为 Adventure Component；hover、URL、COPY 和 callback 在渲染阶段生成。列表状态统一由 `PageSlice` 计算。

## 8. CUI

CUI Layout 为 1-6 行，每行恰好 9 个 ASCII symbol。symbol 支持：

- `Display`：`material/name/lore/amount/custom-data/item-model/glow`。
- `Clicks`：`left/right/shift-left/shift-right/middle/drop`。
- `permission`、`capability`、`hidden-when-disabled`。
- 动态 `source`、`Empty-Display` 和 `{entry.material}`。

约束与生命周期：

- 每个菜单最多一个动态 source symbol，分页容量由其槽位数决定。
- 动态 Material 运行时仍经 `Material.matchMaterial` 校验。
- capability 只允许 `group-title`、`residence-migration`。
- Inventory 点击和拖拽全部取消，不能取出菜单物品。
- holder 绑定玩家、route、revision 和 slot callbacks。
- `OPEN`、`REFRESH`、`STOP` 和失败恢复延迟 1 tick。
- 刷新前确认玩家仍打开同一路由 holder。
- 聊天输入开始前关闭 CUI。

## 9. DUI / Dialog

### 9.1 中立模型

core 使用 `DialogMenuDefinition`、`DialogBodyDefinition`、`DialogInputDefinition`、`DialogButtonDefinition`、`DialogPromptDefinition` 和 `DialogDynamicDefinition`。

支持 KaMenu 风格的 `Title`、`Settings`、`Variables`、`Body`、`Inputs`、`Dynamic`、`Bottom`。不支持 Events、条件、JavaScript 和 Tasks。

### 9.2 Body 与动态列表

Body：

- `message`：文本、宽度、inline buttons。
- `item`：`material/amount/description/width/height/show_overlays/show_tooltip`。

Dynamic：

- page size 为 1-32，每个 entry 最多 4 个按钮模板。
- 当前页 entries 合并进同一个 message Body，以换行分隔。
- 每行按钮占位符转换为唯一 callback ID。
- Bottom 只保留分页、返回、退出和固定操作。
- 不可用 action group 不注册 callback。

Item Body：

- Paper 使用官方 `DialogBody.item(...)`。
- Spigot 使用 `SpigotItemDialogBody` 输出 Mojang `minecraft:item` codec JSON，不直接链接 NMS。
- 当前只支持基础 Material，不支持 name、lore、components、player head 或 slot reference。

### 9.3 Inputs 与二级表单

原生输入类型为 `input`、`slider`、`dropdown`、`checkbox`。input key 使用 snake_case，只允许 `[a-z0-9_]` 风格。

按钮支持：

- `confirmation`：服务端可信确认。
- `input-capture`：单个原生输入。
- `tooltip`：单行或多行悬停文本。

已接入原生输入的业务：

- 创建领地。
- 领地重命名、进入消息、离开消息和地图颜色。
- 创建/重命名权限组。
- 创建/重命名模板。

删除领地、权限组、模板和批量 Residence 迁移等危险操作使用二级确认。尺寸输入仍使用聊天 workflow。

### 9.4 生命周期

- `Bottom` 支持 `notice`、`confirmation`、`multi`。
- `after_action` 支持 `NONE` 和兼容读取 `CLOSE`。
- `WAIT_FOR_RESPONSE` 在加载期拒绝。
- 普通回调不先关闭 Dialog。
- 完成后延迟 1 tick 重新解析 provider 并发送表单。
- 只有显式关闭按钮执行 close。
- Paper callback 自身和共享 token 都限制为一次使用、5 分钟有效。

## 10. 输入基底

`InputSchema` 是三种 UI 共用的规范化层。

DUI 从 `DialogResponseView` 或 `PlayerCustomClickEvent` 提取已声明字段；额外字段被拒绝，checkbox 映射为 `on_true/on_false`，结果写入 `submittedInputs` 和 `{input.<key>}`。

TUI/CUI 使用 `ChatInputWorkflow`：

- 单字段聊天输入。
- 默认等待 2 分钟。
- 提交 future 最多 30 秒。
- 非法输入重新提示并保留 route。
- 取消或超时刷新原 surface。
- 同一玩家只保留一个当前 runner。

schema 校验不能替代业务鉴权；action 仍校验长度、名称冲突、权限和目标状态。

## 11. 玩家语言与 UI 偏好

### 11.1 持久化与首次引导

`PlayerDTO.UI_TYPE` 包含 `BY_PLAYER`、`TUI`、`CUI`、`DUI`。`player_name` 表保存 `ui_preference` 和 `language`。

`V2__PlayerLanguage` 添加 `language`，默认 `NONE`。玩家首次执行 `/dominion` 时：

1. 停止正常菜单渲染。
2. 发送欢迎消息和所有已发现语言的可点击文本。
3. 选择语言后显示 TUI/CUI/DUI 选择。

首次引导不依赖尚未选择的 UI 类型。

### 11.2 Locale

- 内置 locale 来自 manifest。
- 外部 locale 从 `plugins/Dominion/uis/<locale>/` 发现。
- `de_de`、`de-DE`、`de_DE` 等规范化为 `de_DE`。
- 自定义语言可只翻译部分菜单，缺失文件回退 `en_US`。
- 无效玩家 locale 先回退全局语言，再回退 `en_US`。
- reload 后新增语言立即进入语言选择菜单。
- 玩家 locale 只覆盖配置化菜单；部分旧通知仍使用全局语言。

### 11.3 UI 选择

- 固定 `Configuration.defaultUiType` 覆盖个人偏好。
- `BY_PLAYER` 读取数据库 `ui_preference`。
- `00000000` 开头的基岩 UUID 强制 CUI。
- callback 导航保持创建时 surface，不因偏好变化跳界面。
- `ui_preferences` 和 `language_preferences` 是统一二级菜单。

## 12. 平台与构建边界

### 12.1 Java 与模块

| 模块 | 工具链 | 用途 |
| --- | ---: | --- |
| root/core/api | Java 17 | 通用逻辑 |
| dialog-paper/dialog-spigot | Java 21 | Dialog adapter |
| versions/v1_21* | Java 21 | 对应 NMS |
| versions/v26、v26_2 | Java 25 | Mojang 26.x NMS |

通用 JAR 包含各版本模块。旧服务器只加载对应旧实现，可在 Java 21 上运行；26.x 必须使用 Java 25+。

- 1.20.1 至 1.21.x NMS 模块以 `reobf` 产物进入最终 JAR。
- v26/v26_2 使用 Mojang-mapped dev bundle，不执行 reobfuscation。
- Dialog adapter 独立于 NMS 模块。

### 12.2 Dialog API

- Paper 使用 Adventure、Paper builder 和 `DialogResponseView`。
- Spigot 使用 Bungee Dialog POJO、Bungee Component 和 `PlayerCustomClickEvent`。
- 共享接口不能出现任一平台专属类型。
- Paper 不能直接复用 Spigot Bungee Dialog adapter。

### 12.3 Spigot 26.2 Toast

纯 Spigot 26.2 不使用 `UnsafeValues.loadAdvancement`，因为它会写 datapack 并重载资源。

`SpigotToastTransport`：

1. 使用 `Advancement.CODEC + JsonOps` 构造 impossible advancement。
2. 发送 `ClientboundUpdateAdvancementsPacket` 添加已完成进度。
3. 10 tick 后发送 removed packet。
4. 不修改服务器 registry，不写文件。

transport 通过反射隔离在 `dialog-spigot`。初始化失败时明确拒绝 Toast，不回退 `UnsafeValues`。Paper 和旧版本暂时保留 Bukkit advancement 兼容路径。

## 13. 安全与维护约束

- 客户端 payload 不作为权限、目标 ID 或 action path 的可信来源。
- Provider 只返回展示数据、可信参数和允许动作集合。
- action 写数据库或缓存前再次鉴权。
- route、input、action、Material、item model、permission 和尺寸需要校验。
- reload 必须原子替换，不能部分更新。
- renderer 复用 action/provider/session，不复制业务逻辑。
- core 公共签名不能引用 Paper、Folia、CraftBukkit、NMS 或 Spigot Dialog 类型。
- 新增方法或类添加说明用途的英文注释。
- 管理员菜单不自动覆盖；schema 变化需要迁移策略。

## 14. 当前已知边界

1. 内置菜单只有 `en_US` 和 `zh_CN`。
2. 玩家 locale 只覆盖配置化菜单，部分通知仍使用全局语言。
3. TUI 每页一个 LIST，CUI 每页一个动态 source。
4. Dialog 不支持 `WAIT_FOR_RESPONSE`、Events、条件、JavaScript 和 Tasks。
5. Dialog Item Body 只支持基础 Material/amount/description。
6. TUI/CUI 尺寸输入仍走聊天捕获。
7. 旧 UI 仍保留为回退。
8. Spigot Item Body 和 packet Toast 仍需目标客户端最终回归。
9. Paper/旧版本 Toast 仍使用临时 Bukkit advancement。
10. 部分 runtime 框架错误文本仍为硬编码英文。
